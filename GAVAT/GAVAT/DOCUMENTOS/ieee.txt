Introducción

Propósito.

El presente documento tiene como propósito definir las especificaciones funcionales, no funcionales y del sistema para el desarrollo de la plataforma web GAVAT Construcciones
Arquitectónicas. Esta plataforma busca facilitar la gestión y al mismo tiempo integra un sistema de comercio electrónico para la venta de productos relacionados con la construcción y el mejoramiento del hogar y otros espacios.
La plataforma será utilizada por clientes que requieran servicios arquitectónicos o productos de construcción, por proveedores que suministren los materiales, y por el administrador principal (el emprendedor a cargo del proyecto), responsable de la gestión del inventario, la verificación de productos, el control de los pedidos y el control de publicaciones. Al combinar comercio electrónico con gestión de servicios especializados, GAVAT busca optimizar la experiencia del cliente, garantizar
transparencia en las transacciones y contribuir al crecimiento sostenible del negocio.

GAVAT será una plataforma web orientada a la venta de materiales de construcción y servicios relacionados. Su propósito es facilitar a los clientes la exploración, selección, compra y gestión de
productos y servicios de manera rápida, segura y centralizada. La plataforma permitirá una experiencia de compra sencilla y completa, además de un control eficiente por parte del administrador.
La plataforma contempla los siguientes módulos:

●	Módulo Web de Clientes

●	Módulo Administrativo

Definiciones, siglas y abreviaturas


Del negocio.

●	Gerente (Rol de usuario): Persona encargada de la administración general del negocio, control de inventario, gestión de precios y supervisión de pedidos. En este caso, corresponde al dueño del emprendimiento.
●	Vendedor (Rol de usuario): Persona que ofrece los productos y servicios de construcción en la plataforma (enchapados, instalaciones, reparaciones). Este rol será asumido directamente por el emprendedor.
●	Cliente (Rol de usuario): Persona que accede a la plataforma para consultar el catálogo de productos, solicitar servicios de instalación o realizar compras en línea.

Del sistema.

●	Administrar: Acción de agregar, modificar, eliminar y consultar la información de un objeto o usuario en el sistema.
●	Administrador del Sistema: Responsable de mantener la plataforma operativa, actualizando productos, gestionando pedidos y garantizando el soporte técnico.
●	Permiso: Parámetro que define si un usuario dispone de acceso a determinadas funciones o secciones del sistema.
●	Pruebas: Proceso mediante el cual se verifica que la plataforma funciona correctamente, garantizando una navegación segura y estable.
 
●	Rol: Conjunto de permisos y responsabilidades asignados a cada tipo de usuario (cliente, administrador).
●	Usuario: Persona que puede acceder a la plataforma mediante un proceso de registro y autenticación (usuario y contraseña).
●	Ancho de Banda: Capacidad de transferencia de datos en la red, expresada en bit/s, Kbit/s, Mbit/s o Gbit/s.
●	Aplicación: Programa informático que permite al usuario realizar una tarea, en este caso, gestionar compras y servicios de construcción.
●	Base de Datos: Conjunto de datos organizados y almacenados sistemáticamente (MySQL en este proyecto).
●	Bootstrap: Framework de diseño frontend que facilita la creación de interfaces web responsivas y estéticamente atractivas mediante componentes reutilizables en HTML, CSS y JavaScript.
●	Browser (Navegador): Aplicación que permite acceder y visualizar la plataforma web (Google Chrome, Firefox, Edge, Safari).
●	CRUD: Acrónimo de Create, Read, Update, Delete (Crear, Leer, Actualizar, Borrar), operaciones básicas de gestión de datos.
●	Dirección IP: Número que identifica de forma lógica a un dispositivo dentro de la red.

●	Django: Framework de desarrollo web en Python que permite crear aplicaciones robustas, seguras y escalables mediante un enfoque de alto nivel.
●	Hosting: Servicio en línea que permite almacenar y publicar la plataforma en Internet.

●	HTML (HyperText Markup Language): Lenguaje de marcado para estructurar el contenido del sitio web.
 
●	HTTP (HyperText Transfer Protocol): Protocolo utilizado para la transferencia de información en la web.
●	JavaScript: Lenguaje de programación usado para agregar dinamismo e interactividad en la interfaz del usuario.
●	Licencia MIT: Licencia de software abierta, comúnmente utilizada en librerías y frameworks de desarrollo.
●	MySQL: Sistema de gestión de bases de datos relacional que almacenará la información de clientes, productos y pedidos.
●	Python: Lenguaje de programación interpretado, versátil y multiplataforma, utilizado para el desarrollo de la lógica de negocio y la conexión con la base de datos en este proyecto.
●	Django: Un marco web Python de alto nivel que fomenta el desarrollo rápido y el diseño limpio y pragmático.
●	Servidor Web: Programa que implementa el protocolo HTTP y permite que los usuarios accedan al sitio.
●	TCP/IP: Conjunto de protocolos que permiten la comunicación entre dispositivos en la red.

●	URL (Uniform Resource Locator): Dirección utilizada para identificar recursos dentro de Internet.
●	Web: Conjunto de páginas interconectadas que conforman el sitio accesible desde Internet.

Descripción general

Perspectivas del producto.

La plataforma GAVAT Construcciones Arquitectónicas es un producto diseñado para operar en entornos web, lo que permitirá su uso remoto, asincrónico y accesible desde cualquier dispositivo con conexión a Internet. El sistema funcionará de forma independiente, sin requerir interacción directa con sistemas externos, salvo las pasarelas de pago y los servicios de mensajería asociados a las entregas.
El producto consta de una aplicación web principal, accesible desde navegadores modernos en computadores y dispositivos móviles, y estará estructurado en módulos que responden a las necesidades del negocio.

Funciones del producto por roles.

La plataforma GAVAT Construcciones Arquitectónicas permitirá realizar las siguientes funciones, organizadas por rol de usuario:
 
Administrador del sistema (Emprendedor).

●	Perfil: Persona encargada de gestionar la plataforma, actualizar productos, servicios y supervisar los pedidos.
●	Acceso: Exclusivo con credenciales de administrador.

●	Funciones:

o	Gestión de cuentas de clientes: Permite agregar, modificar, activar, desactivar o suspender usuarios registrados. Se cuenta con un panel de administración que muestra la lista de clientes y ofrece opciones de edición de datos y permisos.
o	Administración de productos y servicios: El administrador puede agregar, editar o eliminar elementos del catálogo, manteniendo actualizado el inventario y el portafolio disponible. El sistema ofrece un formulario para registrar nombre, descripción, precio, stock e imágenes, además de la opción de eliminar productos o servicios obsoletos.
o	Gestión de instalaciones: Incluye la configuración de costos adicionales de mano de obra, la asignación de fechas de instalación y el seguimiento de su cumplimiento. El módulo integra un calendario y controles de estado para monitorear los servicios solicitados por los clientes.
o	Gestión de pedidos: El administrador puede supervisar y actualizar el estado de los pedidos (pendiente, en proceso, entregado), incluyendo aquellos que requieren instalación. El panel muestra la información centralizada con opción de asignar entregas a servicios de mensajería.
 
o	Gestión de comentarios: Se administra la sección de reseñas de productos y servicios para asegurar que los comentarios sean útiles, claros y respetuosos. El administrador cuenta con un listado de reseñas por producto con opciones para aprobar, editar, rechazar
o eliminar, además de filtros por usuario, fecha o producto.

o	Gestión de publicaciones: Permite crear, editar o eliminar publicaciones que muestren proyectos realizados. El formulario incluye título, descripción, imágenes, fecha y categoría. El contenido aprobado será visible para los clientes en la sección de novedades
o portafolio.

o	Gestión de envíos y logística: El administrador coordina la entrega de pedidos a través de un panel que integra los estados de los envíos (preparado, en tránsito, entregado) y la asignación a mensajerías como Servientrega o Didi.
o	Revisión de estadísticas del sistema: Se ofrece un panel estadístico con métricas clave: usuarios activos, ventas realizadas, productos más solicitados, servicios más contratados, categorías más consultadas e ingresos totales. Esto apoya la toma de decisiones estratégicas.



Cliente.

●	Perfil: Persona interesada en adquirir productos de construcción y/o contratar servicios de instalación o reparación.
●	Acceso: Requiere registro mediante formulario y autenticación (correo electrónico y contraseña).

●	Funciones:
 
o	Registro: El cliente puede registrarse en la plataforma mediante un formulario que solicita nombre de usuario, correo electrónico, número de teléfono y contraseña. Esto le permite acceder a todas las funciones del sistema.
o	Iniciar sesión: El cliente ingresa a la plataforma a través de un formulario con usuario/email y contraseña, lo que le da acceso a su perfil, pedidos e historial de compras.
o	Visualización del catálogo: El usuario puede explorar productos y servicios disponibles en una página que muestra imágenes, nombres, precios, descripciones y disponibilidad.
o	Visualización de publicaciones: El usuario puede acceder a la sección de publicaciones generadas por el administrador, donde se muestran proyectos realizados. Cada publicación incluye título, descripción, imágenes y fecha, lo que permite al cliente estar actualizado e inspirado en posibles compras o contrataciones.
o	Consulta por categorías: Para facilitar la búsqueda, el catálogo permite filtrar productos por categorías (materiales, herramientas, servicios), a través de un menú desplegable o barra de navegación con filtros activos.
o	Carrito de compras: El cliente puede agregar productos al carrito, modificar cantidades, eliminar elementos y ver el costo total de la compra. Cada producto incluye un botón “Añadir al carrito” y el carrito refleja precio unitario, subtotal y total.
o	Solicitud de instalación: Durante la compra, el cliente puede seleccionar si requiere instalación de un producto. Al activar la opción, el sistema suma automáticamente el costo adicional del servicio al total del carrito.
 
o	Gestión de pedidos: Los clientes pueden consultar en todo momento el estado de sus pedidos y servicios solicitados, con actualizaciones en tiempo real (en preparación, en envío, completado).
o	Comentarios y reseñas: El cliente puede dejar comentarios y calificar productos o servicios mediante un formulario con campos de opinión y puntuación (escala de 1 a 5). También puede consultar las reseñas de otros usuarios, que incluyen texto, puntuación y fecha.
o	Métodos de pago y envío: La plataforma ofrece múltiples opciones de pago: Nequi, tarjetas de crédito/débito y pago contra entrega. En cuanto a logística, los envíos se realizan a través de Servientrega o aplicaciones de mensajería como Didi.
o	Factura en línea: El cliente puede visualizar el detalle de sus compras en una sección de facturación que incluye listado de productos/servicios adquiridos, estado del pedido, subtotal, costos de envío, descuentos aplicados y total pagado.
o	Notificaciones: El sistema mantiene informado al cliente sobre el estado de sus pedidos mediante notificaciones por correo electrónico, SMS o dentro de la aplicación (preparación, envío o entrega).



Visitante (No registrado).

●	Perfil: Usuario que accede por primera vez sin tener una cuenta registrada.

o	Decidir si desea registrarse como cliente para acceder a todas las funciones.

RNF_07	Seguridad de Contraseñas	26/08/2025	Alta
Descripción:	Las contraseñas asociadas a los números telefónicos de los usuarios deberán ser encriptadas mediante algoritmos seguros (bcrypt o equivalentes) antes de ser
almacenadas en la Base de Datos. Esto asegura que la información de acceso esté protegida contra intentos de robo o violación de seguridad.
El sistema nunca deberá almacenar contraseñas en texto plano y deberá aplicar políticas básicas de seguridad como longitud mínima de 8 dígitos/caracteres.
